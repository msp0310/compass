# 実装中の確認事項と採用した既定値

ユーザー確認で作業を止めずに進めるため、判断が必要な項目と現在の採用値を記録します。

## 権限

- システム管理者は全案件を参照・編集でき、代理操作を監査ログへ残す。
- チーム責任者はチーム案件を参照し、要員計画とチーム所属を編集できる。
- チーム責任者がガント計画を編集するには、案件側で `owner` または `planner` が必要。
- PM相当の `owner` は案件設定、基準計画、完了・アーカイブを含む全操作が可能。
- PL相当の `planner` は計画を編集できるが、案件設定・基準計画確定・完了はできない。
- `member` は担当タスクの実績、日報、工数、課題、コメント、添付を扱える。
- `viewer` は参照のみ。

## データ

- 職種 `PM / PL / SE / FE / BE / QA` と権限ロールを分離する。
- 未所属案件は仮想チームを作らず `teamId = null` とする。
- 既存データ互換を優先せず、新しいモデルへMigrationする。
- 予定日と実績日は別フィールドで管理する。

## 認証

- ブラウザ保存のBearerトークンを廃止し、HttpOnly Cookieへ移行する。
- Cookie認証ではSameSite、CSRFトークン、ログイン試行制限を併用する。
- 初期パスワード利用者は、通常画面へ進む前にパスワード変更を必須とする。

## 未確定の製品判断

- 外部閲覧者に課題コメントを許可するか。現時点では参照のみ。
- チーム責任者による案件メンバー追加を許可するか。現時点では要員計画のみ。
- 基準計画の承認をPM単独とするか、上長承認を追加するか。現時点ではPM単独。

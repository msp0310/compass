# 課題

案件内のリスク、確認事項、変更依頼、不具合を表形式で管理する画面です。

## 機能概要

- 課題を案件単位で登録できます。
- 状態、優先度、種別、担当者、関連タスク、期限を一覧で確認できます。
- 内容欄はMarkdown形式で入力し、プレビューで確認できます。
- 課題を開くと、本文の表示と返信履歴を確認できます。
- 返信もMarkdown形式で追加できます。
- 将来のGitHub Issues連携を想定し、内部データ構造だけ連携情報を保持できるようにしています。

## 状態の見方

- 未対応、対応中、ブロック、解決、クローズで管理します。
- 高優先度やブロック中の課題は、案件の進行リスクとして扱います。
- GitHub連携は現時点では画面に出していません。同期処理も未実装です。

## 操作方法

- 課題追加で新しい課題を作成します。
- 一覧行または編集ボタンを選択すると、ダイアログで内容を表示できます。
- 表示画面の編集ボタンから、課題名、状態、本文などを変更できます。
- 表示画面の返信欄から、確認事項や対応履歴を追記できます。
- 内容欄では見出し、箇条書き、チェックリスト、引用、コード、リンクを利用できます。
- 関連タスクを設定すると、ガント上の対象タスクへ移動できます。
